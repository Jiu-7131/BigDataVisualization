# 图表设计优化方案

基于四大设计原则对 12 张图表的设计进行系统优化。

---

## 原则 1：先思考再编码 — 前提问题定案

### 1.1 周期判定算法（图 1 → 下游）

**判定框架**：规则引擎（非打分模型），保证可解释性和零外部依赖。

```
经济方向 ← PMI 的 12 月移动均线环比（上行/下行）
通胀方向 ← CPI 同比环比（上升/下降）

复苏 = PMI↑ + CPI↓
过热 = PMI↑ + CPI↑
滞胀 = PMI↓ + CPI↑
衰退 = PMI↓ + CPI↓

社融存量同比 → 确认经济方向（同步验证）
10Y国债收益率 → 确认通胀预期（交叉验证）
```

当社融或国债收益率与 PMI/CPI 信号矛盾时，输出置信度降级。

**输出格式**（所有下游图表依赖）：

```python
{
    "phase": "复苏",          # 复苏/过热/滞胀/衰退
    "confidence": 0.85,       # 0~1，信号冲突时降低
    "drivers": {
        "pmi": "up",
        "cpi": "down",
        "social_financing": "up",
        "bond_yield": "down"
    }
}
```

每月判定写入 `processed/02_macro/macro_cycle.csv`，图 3/9/10 直接读取该 CSV。

### 1.2 因子收益率归属（图 6/7）

**定案**：因子收益率归入第 3 层 `build_factor_attribution.py`，用截面回归法计算。

方法：每月末对全市场个股做 PE/PB/动量/波动率/换手率/市值排序，做多前 30%、做空后 30%，多空组合下月收益 = 该因子当月收益率。

写入 `processed/03_factor/factor_returns.csv`，图 6 读它做因子暴露校正，图 7 读它做收益归因。消除外部依赖。

### 1.3 三角图归一化（图 2）

当前图 2 的实际实现是 2D 散点图（大小盘溢价 × 价值成长溢价的四象限图），并非三元相图。这比三元相图更符合数据特征——大盘价值、大盘成长、小盘三个指数的收益率是独立变量，无法直接放入三角坐标系。

**定案**：保留当前 2D 四象限实现，不改为三元相图。归一化逻辑：

```
每月末取过去 12 个月：
  x = 小盘累计收益 - 大盘累计收益（大小盘溢价）
  y = 价值累计收益 - 成长累计收益（价值成长溢价）

散点着色按 style_quadrant（大盘价值/小盘成长/大盘成长/小盘价值）
散点大小按 style_drift（风格漂移强度）
```

- 需各自图文件的 `generate()` 函数增加 URL 参数解析逻辑（可选，默认为无）

## 原则 3：精准修改 — 改动范围界定

### 不可动（已验证、与 Git 记录一致）
- 四层流水线架构（`build_*.py`）和顺序依赖
- `config.py` 的路径推导和配色常量
- `visualizations/common.py` 的工具函数
- `visualizations/__init__.py` 的注册表机制
- 函数签名（保持 `generate(data)` 接口）

### 可动（每个图自己的 `chartN_*.py`）
- 只修改对应图文件的交互逻辑和可视化元素
- 不修改数据加载逻辑（`load_all()` 已封装）
- 不做跨文件重构

### 精准修改示例
图 4 去掉凸包：只删除凸包计算代码和对应 Plotly trace，不动滑块、不动数据加载、不动年份自动播放。

## 原则 4：目标驱动执行 — 每图单一成功标准

| 图表 | 核心问题（10 秒回答） | 验证方法 |
|---|---|---|
| 图 1 | 当前处于经济周期的哪个阶段？ | 仪表盘指针一眼定位 |
| 图 2 | 过去 N 个月风格向哪个方向漂移？ | 散点轨迹 + 方向标注 |
| 图 3 | 哪些行业在哪些月份持续跑赢/跑输？ | 颜色块一眼识别连胜/连败 |
| 图 4 | 行业的风险-收益特征如何随时间变化？ | 气泡位置变化趋势清晰 |
| 图 5 | 哪些行业在当前阶段关联性最强/最弱？ | 连线密集度 + 群落颜色 |
| 图 6 | 各行业在因子空间中的定位差异？ | 各行业折线的分离度 |
| 图 7 | 行业超额收益主要来自哪些因子？ | 绿柱（正向贡献）最高者 |
| 图 8 | 哪些因子已经过度拥挤？ | 指针是否进入红色区 |
| 图 9 | 不同经济阶段下哪些行业表现更好？ | 箱体中位数位置对比 |
| 图 10 | 当前阶段应配置哪些行业？ | 桑基图最宽流量路径 |
| 图 11 | 极端情景下哪个行业回撤最大？ | 热图红色最深格 |
| 图 12 | 当前哪些指标触发了预警？ | 红色状态灯数量 |

## 实施顺序（分 5 阶段）

```
阶段 A：图 1 → 图 2 → 图 9     （周期判定 + 风格 + 周期对比）
阶段 B：图 3 → 图 5 → 图 6     （行业轮动 + 关联 + 因子空间）
阶段 C：图 4 → 图 7 → 图 8     （风险收益 + 归因 + 拥挤度）
阶段 D：图 10 → 图 11           （配置建议 + 压力测试）
阶段 E：图 12                    （综合预警）
```

每阶段产出可独立验证，前序阶段的 CSV 输出是后一阶段的输入。

## 与原始设计文档的关系

本优化方案不取代 `图表设计文档.md`，而是对其中设计决策的修订和补充。优化涉及：

1. 明确图 1 的周期判定算法和输出格式
2. 明确因子收益率的计算归属
3. 将图 2 从三元相图修正为 2D 四象限图
4. 精简约 10 项冗余交互
5. 界定精准修改的范围边界
6. 为每图定义单一核心问题和验证标准
